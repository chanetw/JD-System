เมื่อผู้ใช้เรียกใช้คำสั่งนี้ ให้ Agent ดำเนินการตามขั้นตอนต่อไปนี้อย่างเคร่งครัด:

## ขั้นตอนการทำงาน (Step-by-Step)
1. **รับข้อมูล (Receive Input):** รับรายละเอียดการเปลี่ยนแปลง (Change Request) หรือฟีเจอร์ใหม่จากผู้ใช้
2. **วิเคราะห์ผลกระทบ (Impact Analysis):** - ตรวจสอบว่ากระทบต่อโมดูลใดใน `modules/` บ้าง
   - ตรวจสอบความขัดแย้งกับกฎธุรกิจเดิม (SLA, Quota, Approval Flow)
   - วิเคราะห์ว่าต้องมีการเพิ่มหรือแก้ไขตารางใน Database (17 ตารางหลัก) หรือ API หรือไม่
3. **ร่างแผนการเปลี่ยนแปลง (Proposed Change Plan):**
   - สรุปสิ่งที่ต้องแก้ไขเป็นภาษาไทย
   - ระบุไฟล์ที่เกี่ยวข้องที่อาจต้องมีการอัปเดต
4. **อัปเดตเอกสาร (Document Update):**
   - เตรียมเนื้อหาที่จะอัปเดตในไฟล์ `REQUIREMENT.md` โดยเฉพาะในส่วน **Change Log**
5. **ขออนุมัติ (Request Approval):** แสดงแผนงานทั้งหมดและถามผู้ใช้ว่า "อนุญาตให้อัปเดตเอกสารและดำเนินการต่อหรือไม่?"

## กฎเหล็ก
- **ห้ามแก้ไขไฟล์ REQUIREMENT.md** จนกว่าจะได้รับคำสั่ง "อนุญาต"
- การอธิบายทั้งหมดต้องเป็น **ภาษาไทย** ที่ชัดเจนและเป็นทางการ
