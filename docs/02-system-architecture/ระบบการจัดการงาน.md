## 1. ระบบจัดการผู้ใช้

### 1.1 ข้อมูลผู้ใช้

**แหล่งที่มา**: `backend/prisma/schema.prisma` (model User)

```
ผู้ใช้งาน
├─ ข้อมูลส่วนตัว
│  ├─ firstName (ชื่อจริง)
│  ├─ lastName (นามสกุล)
│  ├─ displayName (ชื่อแสดง)

│
├─ สถานะผู้ใช้
│  ├─ isActive (เปิดใช้งาน)
│  ├─ status: PENDING | APPROVED | REJECTED | INACTIVE
│  │   └─ PENDING: รอการอนุมัติจากแอดมิน

│  ├─ approvedAt (วันที่อนุมัติ)

├─ สิทธิการเข้าถึง
│  ├─ departmentId (หน่วยงาน)

│  │  ├─ tenantId
│  │  ├─ projectId
│  │  └─ roleType
│  └─ managedDepartments[] (แผนกที่จัดการ)
│
├─ ประวัติการเข้าสู่ระบบ
│  ├─ lastLoginAt (ล็อกอินครั้งล่าสุด)
│  └─ mustChangePassword (ต้องเปลี่ยนรหัสผ่าน)
│
└─ ความสัมพันธ์
   ├─ requestedJobs[] (งานที่ร้องขอ)
   └─ assignedJobs[] (งานที่รับผิดชอบ)
```

### 2.1 โครงสร้าง Approval Flow

**แหล่งที่มา**: `backend/prisma/schema.prisma` (model ApprovalFlow)

```
เทมเพลตการอนุมัติ
├─ ระบุ
│  ├─ projectId (โครงการ) *
│  ├─ jobTypeId (ประเภทงาน) - NULL = Default
│  ├─ name (ชื่อเทมเพลต)
│  └─ description (คำอธิบาย)
│
├─ การไม่อนุมัติ
│  ├─ skipApproval (BOOL)
│  │  └─ true = ข้ามการอนุมัติทั้งหมด
│  └─ autoAssignType
│     ├─ manual (เลือกเอง)
│     ├─ team_lead (หัวหน้าทีม)
│     ├─ dept_manager (หัวหน้าแผนก)
│     └─ specific_user (ผู้ใช้เฉพาะ)
│
├─ การมอบหมายอัตโนมัติ
│  └─ autoAssignUserId (ถ้า specific_user)
│
├─ ระดับการอนุมัติ
│  └─ approverSteps (JSON Array)
│     ├─ Level 1: [ ผู้อนุมัติคน A, คน B ]
│     ├─ Level 2: [ ผู้อนุมัติคน C ]
│     └─ Level 3: [ ผู้อนุมัติคน D ]
│
├─ การควบคุมอื่น
│  ├─ allowOverride (อนุญาตปฏิเสธ)
│  ├─ effectiveFrom (วันเริ่มใช้)
│  └─ effectiveTo (วันสิ้นสุด)
│
└─ ข้อจำกัด
   └─ UNIQUE [projectId, jobTypeId]
      └─ projectId + NULL = Default Flow
```

## 3. การสร้าง DJ ID

### 3.1 รูปแบบ DJ ID

**นิยาม:**

```
DJ-YYMMDD-XXXX

โดยที่:
  DJ    = คำนำหน้าคงที่ (สตริง "DJ")
  YY    = ปีค.ศ. (2 หลักสุดท้าย) → 26 สำหรับ 2026
  MM    = เดือน (2 หลัก, มีศูนย์นำ) → 01-12
  DD    = วันที่ (2 หลัก, มีศูนย์นำ) → 01-31
  XXXX  = เลขลำดับรายวัน (4 หลัก, มีศูนย์นำ) → 0001-9999
```

**ตัวอย่าง:**

```
วันที่ 6 กุมภาพันธ์ 2026
├─ DJ-260206-0001 (งานแรกของวัน)
├─ DJ-260206-0002
├─ DJ-260206-0003
└─ DJ-260206-9999 (งานที่ 9999 ของวัน)

วันที่ 7 กุมภาพันธ์ 2026
└─ DJ-260207-0001 (เลขลำดับรีเซตแล้ว)
```

### 3.2 ลอจิกการสร้าง DJ ID

**แหล่งที่มา**: `backend/api-server/src/routes/jobs.js` (lines 313-328)

**ขั้นตอน:**

```javascript
// ขั้นที่ 1: สร้าง Date Prefix
const now = new Date();
const yy = String(now.getFullYear()).slice(-2);      // 26
const mm = String(now.getMonth() + 1).padStart(2, '0'); // 02
const dd = String(now.getDate()).padStart(2, '0');     // 06
const datePrefix = `DJ-${yy}${mm}${dd}-`;            // DJ-260206-

// ขั้นที่ 2: นับจำนวนงานที่มี Prefix เดียวกัน
const jobCount = await prisma.job.count({
  where: {
    tenantId,                    // แยกต่างคนต่างแรนต์
    djId: { startsWith: datePrefix }
  }
});
// jobCount = 0 (งานแรกของวัน)

// ขั้นที่ 3: สร้าง DJ ID
const djId = `${datePrefix}${String(jobCount + 1).padStart(4, '0')}`;
// djId = DJ-260206-0001
```

### 3.3 งาน Parent-Child (ความสัมพันธ์ระหว่างงาน)

**รูปแบบ:**

```
Parent Job:     DJ-260206-0001
├─ Child 1:     DJ-260206-0001-01
├─ Child 2:     DJ-260206-0001-02
└─ Child 3:     DJ-260206-0001-03
```

**ลอจิก:**

```javascript
// 1. สร้าง Parent DJ ID ตามปกติ
const parentDjId = `DJ-260206-0001`;

// 2. สร้าง Child DJ ID ด้วย suffix
const generateChildDjId = (childIndex) => {
  return `${parentDjId}-${String(childIndex + 1).padStart(2, '0')}`;
};

// 3. ผลลัพธ์
generateChildDjId(0) // DJ-260206-0001-01
generateChildDjId(1) // DJ-260206-0001-02
generateChildDjId(2) // DJ-260206-0001-03
```

### 3.4 คุณสมบัติ DJ ID

| คุณสมบัติ | รายละเอียด |
|---------|----------|
| **ความเป็นเอกลักษณ์** | Unique ต่อ Tenant (ไม่ซ้ำกันในผู้เช่า) |
| **การแบ่งตามวัน** | Prefix เปลี่ยนทุกเที่ยงคืน UTC |
| **ลำดับต่อเนื่อง** | ลำดับเพิ่มทีละ 1 ต่อวัน (0001-9999) |
| **ขอบเขต Tenant** | Tenant ต่างกัน → ตัวเลขเดียวกันได้ |
| **ไม่เปลี่ยนแปลง** | ไม่สามารถอัปเดตหลังสร้างได้ (Immutable) |
| **ดัชนี** | Indexed ใน Database สำหรับประสิทธิภาพ |
| **เก็บข้อมูล** | ชนิด: VARCHAR(50) |

```
│ 9,999 งาน/วัน       │ ← Max XXXX = 9999

  → DJ-260206-9999 ← งานสุดท้ายของวัน

```
