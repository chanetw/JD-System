feat: Remove V2 Approval Flow, Extend V1 with Job Type + Skip Approval

BREAKING CHANGE: Removes V2 approval flow template system entirely.

## Summary

Removed the V2 approval flow template system and extended V1 to support:
- Job Type-specific approval flows (with fallback to default)
- Skip Approval feature (bypass approval steps entirely)
- Auto-assignment options (manual, dept_manager, team_lead, specific_user)

## Changes

### Database
- Migration 016: Extend V1 columns + Drop V2 tables (with archive backup)
  - Added: job_type_id, skip_approval, auto_assign_type, auto_assign_user_id
  - Created: Performance indexes for (project_id, job_type_id) lookups
  - Dropped: V2 tables (approval_flow_templates, steps, assignments, approvers)
  - Backup: All V2 data in *_archive tables (for rollback)

### Prisma Schema
- ApprovalFlow: Added 4 new fields + 2 relations
- Removed: 4 V2 models (ApprovalFlowTemplate, Step, Assignment, Approver)
- Updated: Tenant, User, Project, JobType relations

### Backend Services
- approvalService.js:
  - New: getApprovalFlow() with fallback logic
  - New: isSkipApproval(), getApprovalLevels(), autoAssignJob()
  - Removed: 5 V2 methods (getFlowAssignmentV2, etc.)
  - Updated: approveJobViaWeb() to use V1 extended methods

- jobs.js:
  - Updated job creation to use V1 extended flow resolution
  - Supports skip approval flow directly

- Removed: approval-flow-templates.js routes (V2)

### Frontend
- ApprovalFlow.jsx:
  - Added Job Type Selector (radio buttons)
  - Added Skip Approval Toggle
  - Added Auto-Assign Configuration (shown when skip=true)
  - Conditional rendering: Approval steps only when skip=false
  - Updated handleSaveFlow() to include V1 extended fields

- Removed: ApprovalFlowTemplates.jsx (V2 page)

## Impact

### Positive
- Simpler system: V2 complex template system removed
- Faster: Single table lookup instead of 4-table join
- Cleaner: Fewer models, fewer code paths
- Flexible: Support different workflows per job type

### Breaking Changes
- V2 API endpoints removed: /api/approval-flow-templates
- V2 UI removed: /admin/approval-templates
- Cannot create/edit approval flow templates via API anymore (UI only)

### Migration Required
- Must run: 016_extend_v1_remove_v2_approval_flow.sql
- Data preserved: V2 data in archive tables
- Rollback: Use 016_ROLLBACK_extend_v1_remove_v2.sql if needed

### Testing
- All smoke tests passing
- Build verified (frontend + backend)
- Schema valid
- Performance indexes created
